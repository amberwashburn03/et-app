Core Objectives
The main goals driving this project
  • →
    Voice-first food logging — Users should be able to say what they ate naturally, with no typing required. Designed for real-world use while cooking, driving, or working out.
  • →
    Personality over guilt — ET roasts junk food, celebrates goals, and calls out skipped workouts with humor. Escalating reactions, never shame-based like Duolingo's sad owl.
  • →
    No scale needed — Portion intelligence using hand method (palm, fist, thumb) and visual sizes. Designed for normal people, not bodybuilders weighing every gram.
  • →
    Family app — Built for the Washburn household. Future family mode planned with household dashboard for Trip, Gage, Chris, Seth, and Reagan.
  • →
    Walmart / Sam's Club integration — Pantry auto-populates from those stores. Receipt scanning reads brands for accurate calorie lookup.
  • →
    Installable on Android — Must feel like a real downloaded app with no browser bar. PWA on GitHub Pages satisfies this without app store submission.
  • →
    Accent-resilient voice — Claude Haiku accent cleanup agent runs on every voice transcript before food lookup. Fixes mishearing, spoken numbers, filler words.

Features Built
All confirmed working in current index.html
🎤 Voice — Press & Hold
Hold orb to record, release to process. Words appear live in input box as you speak. Continuous SpeechRecognition mode. Wake word loop separate from tap flow.
👽 Accent Cleanup Agent
Claude Haiku post-processes every voice transcript. Fixes "brokely" → "broccoli", "two eggs" → "2 eggs", strips filler words. Falls back gracefully if API unavailable.
🗣️ Wake Words
Continuous loop listens for: "Hey ET", "E.T.", "Hey Eat" + 10 accent/mishearing variants. Inline command support: "Hey ET I had 3 eggs" logs instantly in one phrase.
📳 Haptic Feedback
Short buzz (40ms) on mic press. Double pulse (60/40/60ms) on wake word. Light double tap (30/20/30ms) on food logged. Silent on iOS/desktop.
🍽️ Food Database (400+ items)
Instant local dictionary — Smucker's, Rold Gold, Ben & Jerry's, energy drinks, fast food chains. 4-tier lookup: pantry → local DB → Claude Haiku → generalFallback().
💧 Color-Coded Drink Tracker
Each drink type = unique color + glow. Energy = neon green fizz, Coke = dark fizz. No 8-glass cap — shows all past goal with orange border.
⚡ Live Goal Ticker
4 cards: calories, protein, water, carbs. Always uses effectiveGoals() — never raw goals.*. Turns green + pops animation on hit. Workout adjustment badge shown.
🏋️ Devices + Workout Import
6 wearables (UI built). Screenshot import via Claude vision. Text parser: Keytel HR formula, StairMaster METs. 5 intensities auto-adjust macros. Sweat log with delete.
🛒 Pantry Profile
Receipt upload (photo or text). AI reads brands. Manual add. Store chips for Walmart/Sam's. Pantry items highest priority in food lookup chain.
📊 Meal Time Insights + Reminders
Tracks calories by day-of-week + time slot. Fires pattern insight if 60%+ of instances exceed 40% of daily goal. User can save reminder that fires every matching slot.
✋ Portion Intelligence
Hand method, visual sizes, containers. Gym defaults only fire when portion IS specified. "I had 3 medium over easy eggs" correctly returns 3 eggs.
🧠 Smart Quick Log
Time-of-day suggestions. Learns personal habits with frequency scoring. Custom recipes pinned to top. Shows ET personalization badge once habits are learned.
⚙️ Goals + Body Profile
Cal/protein/carbs/fat/water steppers. Weight, age, sex for HR calorie formula. Goals persist across page refreshes via localStorage 'et_goals'. ET Settings section.
🔔 Check-In System
Contextual meal-slot nudges. "It's 2pm — you need 45g protein still." Per-slot, once per day, never repeats. Fires only in second half of each slot window.

Critical Architecture Rules
Never break these — they protect data integrity
RULE 1 — NEVER use goals.* directly for display or calculations. Always call effectiveGoals() which adds workout adjustments on top. effectiveGoals() = goals + goalAdjustments (persisted to 'et_goal_adj') RULE 2 — FOOD LOOKUP ORDER (always this exact chain): 1. pantryItems (localStorage 'et_pantry') — instant, highest priority 2. FOOD_DB dictionary — instant, 400+ entries 3. Claude Haiku via fetch with 20s AbortController timeout 4. generalFallback() — always returns something, never blocks user RULE 3 — DRINK LOGGING — every drink must push to ALL THREE: entries[] (main food log — shows in Today's Log) drinkEntries[] (drink subtotal tracker) glassLog[] (color glass tracker) matchDrinkType() maps brand names → DRINKS color entries RULE 4 — VOICE ENGINE (press-and-hold): orbDown() → stops wakeRecognition → 150ms delay → _startContinuousListen() orbUp() → _stopAndClean() → _cleanAndProcess() → processFood() continuous: true, interimResults: true (words show live in input box) Errors 'no-speech' and 'aborted' are silently ignored in continuous mode RULE 5 — API KEY storage key is lowercase: localStorage 'et_api_key' ← correct localStorage 'ET_api_key' ← WRONG — causes key to vanish on reload Migration code auto-copies from old uppercase key if found

Voice System — Complete Reference
How the full voice pipeline works end to end
  • 1
    User presses and holds the orb — orbDown() fires. Haptic buzz (40ms). Any running wakeRecognition is stopped first (browser only allows one SpeechRecognition at a time). 150ms delay lets browser release the mic.
  • 2
    _startContinuousListen() runs — Creates SpeechRecognition with continuous: true, interimResults: true. Words appear live in the food input box as you speak. The orb glows green and label says "🔴 hold & speak".
  • 3
    User releases the orb — orbUp() fires. Haptic double tap (30/20/30ms). _stopAndClean() stops recognition and grabs the full transcript.
  • 4
    Accent cleanup agent runs — _cleanAndProcess() sends raw transcript to Claude Haiku with a 7-second timeout. Prompt fixes mishearing, converts spoken numbers, strips filler words, corrects food names. Result shown in input box.
  • 5
    processFood() runs on cleaned text — Runs the 4-tier food lookup chain. Logs the entry. ET reacts. Haptic success buzz (30/20/30ms). Wake word loop resumes after 4 seconds.
Wake word phrases (all trigger the mic): Primary: 'hey et' · 'e.t.' · 'hey eat' Variants: 'hey e.t.' · 'hey e t' · 'hey eighty' 'hey eight' · 'hey ate' · 'okay et' 'ok et' · 'a et' · 'hey it' 'hey eats' · 'hey eaten' · 'hey eater' Inline command: "Hey ET I had 3 eggs" → logs "3 eggs" immediately, no second tap needed Haptic patterns: 40ms → mic activated (orb pressed) 60/40/60ms → wake word confirmed 30/20/30ms → food logged successfully

PWA / GitHub Pages Deploy
How to host and install ET on Android
⚠️ Critical Lessons Learned
Blob URL manifests are blocked by Chrome. Blob URL service workers are blocked by Chrome. Icons must be real PNG files at correct declared sizes. sw.js MUST be a real separate file — no workarounds exist.
✅ What's Working Now
5 real files: index.html + sw.js + manifest.json + icon-192.png + icon-512.png. Auto-update bumper timestamps sw.js on every build. App reloads itself on new version detection.

All Bugs Fixed Across All Sessions
Full history from session 1 through today
Syne font never imported
CSS and pattern insights HTML both used font-family: 'Syne' but it was never in the Google Fonts link. Added to import.
Goals wiped on every page refresh
saveGoals() updated memory but never called localStorage.setItem. Goals also weren't loaded on startup. Now persist to 'et_goals'.
ET_api_key vs et_api_key case mismatch
Key was saved uppercase but loaded lowercase — different slots. Silently lost on every reload. Fixed + migration code copies old key automatically.
clearTimeout was a no-op in catch block
timeout declared const inside try → out of scope in catch. clearTimeout(undefined) called. Moved declaration outside try block.
Smart chips never rendered
renderSmartChips built the chips array but never mapped it to HTML before prepending recipes. Container stayed empty or stale.
hover-bg invalid CSS property
hover-bg was not a real CSS property. Silently ignored. Removed.
Voice used MediaRecorder + Claude audio
Original design tried to send audio to Claude API for transcription. Claude doesn't accept audio. Replaced with Web Speech API directly.
Wake word conflicted with orb tap
Browser allows only one SpeechRecognition at a time. wakeRecognition running when orb tapped → instant abort error. Now stopped explicitly before starting new session.
continuous: false caused "voice error"
Browser auto-stopped after first silence before finalizing result. Fell through to error handler. Changed to continuous: true + press-and-hold model.
Blob URL manifest / service worker blocked
Chrome blocks both. Multiple sessions tried this as a "single file" workaround. Now using real separate files — no alternative exists.
Icons wrong dimensions
72px image declared as 192px and 512px in manifest. Chrome validates actual size. Generated real alien-face PNG icons at correct sizes.
saveGoals hardcoded goals.cal
Workout bumps reset on every save. Fixed to use spread + effectiveGoals().
logDrink buttons missing from Today's Log
Drinks weren't being pushed to entries[]. Fixed — all three arrays now updated.
todayJunkCount never reset at midnight
Fixed with date comparison on et_count_date localStorage key.
goalAdjustments lost on page refresh
Workout bumps vanished on reload. Now persisted to 'et_goal_adj'.
NaN in macro totals
Null guards added to all addEntry/deleteEntry math.

Key Decisions Made
Finalized preferences and design choices
  • ✓
    Press-and-hold mic, not tap-to-toggle — You preferred the physical feel of holding while talking and releasing when done. Matches how walkie-talkies and voice memos work.
  • ✓
    Roast-y personality, never guilt — ET is a funny sidekick. Reactions escalate with junk food (1st/2nd/3rd+). Redemption arcs when healthy food follows junk. No sad animals, no negative shame loops.
  • ✓
    Accent cleanup agent is always on — Every voice transcript goes through Claude Haiku for cleanup before food lookup. This handles different accents, mumbling, spoken numbers, and filler words.
  • ✓
    Single HTML file architecture — Entire app in one file for simplicity. PWA support requires sw.js, manifest.json, and icon files as separate real files — no blob URL workarounds.
  • ✓
    GitHub Pages for hosting — Free, HTTPS (required for mic), no backend needed. Repo is public. Live URL format: https://[username].github.io/et-app
  • ✓
    Auto-update via sw.js timestamp — Every time a new index.html is deployed, sw.js gets a fresh BUILD timestamp. Chrome detects the change and pushes the update to all installed users silently.
  • ✓
    No app store — PWA only — Install directly from Chrome on Android. No Google Play submission needed. Samsung S25+ tested and confirmed working via "Install App" in Chrome menu.
  • ✓
    API key asked once only — Stored in localStorage 'et_api_key' (lowercase). Migration code handles users who saved it under the old uppercase key. Banner hides once key is saved.
